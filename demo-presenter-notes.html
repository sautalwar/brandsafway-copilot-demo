BrandSafway · Developer Agent Demo

Repeatable agentic AI demo sequence

Goal: move one business request from Azure DevOps to shipped code while showing GitHub Copilot across VS Code, CLI, and the coding agent. Keep the story practical: same workflow, same controls, faster execution.

1 ADO
2 Issue
3-7 AI Work
8-9 Review/Gates
10 Merge
Anchor line: BrandSafway keeps Azure DevOps and GitHub exactly as-is; agentic AI augments the implementation step and stays inside the normal branch, PR, review, and pipeline controls.

Demo path options

Recommended for live demos: Use Path B for reliability and fewer moving parts. Use Path A in production when you want full ADO-to-GitHub automation.

Path A — Full Automation

ADO Service Hook → ado-issue-sync.yml → automatic issue creation + Copilot assignment.

Setup guide: docs/setup-ado-sync.html

Path B — Manual + Auto-Assign

Create the ADO work item manually → create the GitHub issue manually with AB# + copilot-agent label → auto-assign-copilot.yml handles Copilot assignment.

Setup guide: docs/setup-auto-assign.html

Pre-demo setup checklist

Environment

  • Sign in to Azure DevOps, GitHub, VS Code, and GitHub Copilot.
  • Open repo in VS Code with Copilot Chat visible.
  • Open terminal with gh copilot ready.
  • Confirm issue assignment permissions for @copilot.
  • Have GitHub Actions / ADO pipeline tab preloaded.

Browser layout

  • Monitor 1 projected: ADO, GitHub, VS Code, pipeline tabs only.
  • Monitor 2 private: this notes file.
  • Tabs ordered: ADO board → GitHub issues → branch/PR → pipeline.
  • Zoom browser to 110–125% for room readability.
  • Disable notifications and unrelated tabs.

Backup assets

  • One pre-created ADO item tagged copilot-demo.
  • One pre-created GitHub issue with AB# reference.
  • Prepared branch, commits, PR, approval, and green CI.
  • Screenshots of assignment, PR, and pipeline in case latency hits.

Opening frame

“You’re about to watch a normal backlog item go from Azure DevOps into GitHub, get implemented with Copilot, reviewed by a human, validated by the same pipeline, and merged with a full audit trail.”

Key talking points

Agentic AI value

  • Issue-driven execution, not isolated prompting.
  • Works inside branches, commits, PRs, reviews, and pipelines.
  • Augments developers across IDE, CLI, and autonomous agent surfaces.
  • Best for scoped, reviewable work with crisp acceptance criteria.

Common objections to pre-answer

  • Security: same repo permissions and gates.
  • Quality: tests + review + CI still mandatory.
  • Governance: ADO remains the planning/reporting layer.
  • Audit trail: AB#, issue, branch, commits, PR, pipeline, merge.

Wrap-up: “The outcome is faster implementation without giving up control, traceability, quality, or security.”
